SESI 2: IMPLEMENTASI STRUKTUR DATA MURNI (PURE DATA STRUCTURES)
----------------------------------------------------------------------
[Context / Problem]
- Melengkapi seluruh `# TODO` kode pemrograman pada halaman 41-42 panduan laboratorium.
- Larangan penggunaan struktur data bawaan (built-in list/dict) untuk fungsionalitas utama dan kewajiban enkapsulasi berbasis pointer Linked List.

[User Prompt]
- "lengkapilah todo code tersebut dan buatkan juga data_structures bst.py graph.py linked_list.py queue_11.py stack.py ... "

[AI Solusi & Implementasi]
- `linked_list.py`: Membangun simpul rantai `LLNode` sebagai fondasi data linear bertipe objek.
- `queue_ll.py`: Mengimplementasikan `PriorityQueueBantuan` dengan metode `.enqueue()` berbasis *insertion sort* terurut prioritas $O(n)$ (Prioritas level=1 KRITIS didahulukan) dan `.dequeue()` efisiensi konstan $O(1)$.
- `stack.py`: Merealisasikan struktur `Stack` murni (LIFO) untuk mencatat log pengiriman, dilengkapi metode `.to_list()` untuk mengekstraksi data dari *top* ke *bottom* demi kebutuhan audit kronologis.
- `bst.py`: Membuat struktur `BSTLokasi` dan `BSTNodeLok` dengan kunci `lokasi.kode` untuk mendukung operasi `.insert()`, `.search()`, dan `.update_level()` dengan kompleksitas rata-rata $O(\log n)$.
- `graph.py`: Menyelesaikan kelas `EdgeNode` dan `GraphRute` menggunakan representasi *Adjacency List* kustom untuk graf tidak berarah. Melengkapi fungsi `.bfs_akses()` untuk mengembalikan data keterjangkauan node dari depot.
